PR: 平面像素星空可视化基础实现（Receiver Visual Mock v0.1）
目标

实现 Sputnik 首页的核心视觉层：一张平面像素化宇宙星图。

这张星图不是普通背景，而是页面的主要信息界面。
其中：

大部分星点代表广播乐谱中的音符
音乐播放推进时，对应音符星点会被点亮
少量固定恒星代表个人项目入口
整体视觉风格为低分辨率、稀疏发光、冷色、黑暗宇宙

本 PR 聚焦纯可视化基础能力，不接入真实生成链路。

范围
本 PR 要做
建立全屏平面星空画布
实现广播音符星点的静态渲染
实现项目恒星的静态渲染
实现广播星点的时间推进点亮逻辑
实现基础 hover 态的项目星交互反馈
完成整体像素化宇宙视觉基调
本 PR 不做
不接真实 Qwen 输出
不接真实 note-json 协议最终版
不接真实音频播放同步
不接真实项目跳转逻辑
不接污染输入逻辑
不做移动端最终适配收口
设计原则
1. 星空不是背景，是主界面

星图承担主要信息表达，而不是装饰层。

2. 星点不是随机粒子，而是结构化节点

广播星点有数据来源，项目星点有固定语义。

3. 像素化不是复古游戏感，而是低分辨率宇宙信号感

发光应稀疏、克制、偏硬边，不做柔滑梦幻粒子。

4. 项目入口应像“被命名的恒星”，不是按钮

需要可识别，但不能破坏宇宙沉浸感。

可视化结构
A. 广播音符星层

作用：表示当前广播中的音符节点。

特征：

数量最多
分布于平面星空主区域
默认低亮
播放推进时被点亮
不可点击
由 mock score 数据驱动
B. 项目恒星层

作用：表示固定存在的项目入口。

特征：

数量少
位置固定
默认较稳定
可 hover
hover 后显示极小标签
后续可点击跳转
视觉上比普通广播星更稳定，但不夸张
C. 背景噪声层

作用：补充空间深度和宇宙氛围。

特征：

最暗
最稀疏
不承担明确信息语义
不能抢戏
平面星空映射逻辑（方案 B）

本 PR 使用 固定平面宇宙映射，而不是图表式时间-音高坐标图。

广播音符星点

每个音符映射为平面中的一个固定星位。

建议映射原则：

相近音高落在相近区域
不同 track 分布在不同星域
相邻时间片段在局部区域内形成“星群流动感”
同类音符在视觉上具有一定聚类趋势

目标不是精确乐谱展示，而是：

把广播结构映射成一张可感知、可点亮的宇宙平面。

项目恒星

固定配置坐标，例如：

Sputnik
ItClicks
DriftProbe
其他项目

要求：

与广播星层共存
不与活跃音符区完全重叠
建议位于主星群外围或边缘稀疏区
视觉规范
背景
纯黑或极深蓝黑
保持空间留白
禁止复杂渐变背景抢戏
广播音符星

默认状态：

小尺寸
低亮度
像素块/十字/小方点风格
冷白、灰蓝、极淡青

激活状态：

明度明显提升
可带短余辉
可略微外扩
不能过于梦幻或模糊
项目恒星

默认状态：

比普通广播星略稳定
亮度略高
尺寸可略大半级

hover 状态：

星点增强
周围出现极淡像素环或微弱脉冲
出现极小标签文字
背景噪声星
极暗
尺寸最小
不参与动画主节奏
动效规范
1. 广播点亮

基础逻辑：

按时间推进激活对应音符星点
点亮后保留短暂余辉
消退后回到低亮状态
2. 轻微整体漂移

可选非常弱的整体漂移/呼吸感，模拟接收器工作状态。
要求几乎不可察觉。

3. 项目恒星脉冲

项目星可以有低频、稳定、缓慢的脉冲感。
禁止闪烁过于明显。

4. hover 对焦反馈

hover 项目星时，应表现为“锁定一个远方坐标”，而不是普通网页按钮 hover。

数据输入（mock）
广播星 mock 数据

需要一份最小可视化数据，例如：

{
  "notes": [
    { "id": "n1", "t": 0, "d": 1, "p": 60, "v": 0.8, "track": 0 },
    { "id": "n2", "t": 1, "d": 1, "p": 64, "v": 0.6, "track": 1 },
    { "id": "n3", "t": 2, "d": 2, "p": 67, "v": 0.7, "track": 0 }
  ]
}
项目星 mock 数据

例如：

{
  "projects": [
    { "id": "sputnik", "label": "Sputnik", "x": 0.78, "y": 0.22 },
    { "id": "itclicks", "label": "ItClicks", "x": 0.18, "y": 0.64 },
    { "id": "driftprobe", "label": "DriftProbe", "x": 0.84, "y": 0.71 }
  ]
}

说明：

x / y 使用归一化坐标
项目星位置固定
广播星位置可由初始化映射函数生成

点数量可控

像素化风格更容易控制
比 DOM 星点方案更稳

如果后续需要复杂交互，再评估：

WebGL / PixiJS

但 v0.1 没必要。
